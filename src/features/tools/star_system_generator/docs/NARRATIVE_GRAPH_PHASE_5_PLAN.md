## Architectural Notes

### Why historical edges matter for spine summaries

Phase 4's deep audit reported `spineSummary` length tightly clustered at 111–116 chars across p10/p50/p90. That clustering is the symptom of a spine summary that always uses one template variant from one edge family. The master plan's worked example is ~190 chars and weaves a historical clause:

> "Orison Hold has held the chiral ice belt since the second-wave terraforming failure forced the survivors onto it — but the Route Authority and the Kestrel Free Compact can't both set the quota."

Phase 5 widens the length distribution by giving the renderer a real second clause to weave when the top spine edge has a linked historical edge.

### Edge-attachment scoring

Per the master plan, score "needs backstory" on each spine edge:

| Spine edge type | Score | Rationale |
|---|---|---|
| `CONTROLS`, `CONTESTS`, `DEPENDS_ON`, `CONTRADICTS` | high (1.0) | "why is this the case?" is the obvious reading question |
| `DESTABILIZES`, `SUPPRESSES` | medium (0.6) | the past act that started it carries dramatic weight |
| `HOSTS`, `WITNESSES` | low (0.0) | self-explanatory; no backstory needed |

Sort spine edges by score descending, then attach historical edges to the top ones. Cap at 2 historical edges per system. Sparse systems may produce 1 or 0.

### Present-to-historical type map

Per the master plan, each present-edge type has exactly one eligible historical type:

| Present | Historical | Rendering intent |
|---|---|---|
| `CONTROLS` | `FOUNDED_BY` | "X founded the controlled entity during era Y" |
| `CONTESTS` | `BETRAYED` | "A and B's compact broke during era Y" |
| `DEPENDS_ON` | `DISPLACED` | "X was displaced to its current dependency during era Y" |
| `DESTABILIZES` | `FOUNDED_BY` | "the destabilizing entity was founded under flawed assumptions" |
| `SUPPRESSES` | `BETRAYED` | "a compact was broken when the suppressing party seized control" |
| `CONTRADICTS` | `BETRAYED` | "records were edited as a breach of public trust during era Y" |

`HOSTS` and `WITNESSES` are not eligible (their score is 0 above).

### Subject/object selection for historical edges

The historical edge's subject and object are derived from the present edge:
- `FOUNDED_BY`: subject = present edge's controlling actor (subject of CONTROLS / DESTABILIZES); object = the controlled / destabilizing entity (object of the present edge).
- `BETRAYED`: subject = the actor that did the breaking (subject of CONTESTS / SUPPRESSES / CONTRADICTS); object = the actor or institution betrayed (object of the present edge).
- `DISPLACED`: subject = the displaced population (the dependent — subject of DEPENDS_ON); object = the resource or location they now depend on.

These are starting points. The implementer can adjust if a present-edge instance produces nonsense subjects (e.g., a `CONTRADICTS` whose object is `system` — the historical edge's object should fall back to a more concrete entity if available).

Era selection inside `attachHistoricalEvents` uses `ctx.rng.fork('history').next()` to pick an index — deterministic per seed, independent of present-edge RNG draws.

### `{historical:summary}` and `{historical:era}` slot resolution

Phase 3 returns empty string for `{historical:*}`. Phase 5 changes that, but only when the renderer passes historical context.

The cleanest design extends `EdgeRenderContext` with an optional field:

```ts
export interface EdgeRenderContext {
  subject: EntityRef
  object: EntityRef
  qualifier?: string
  edgeType: EdgeType
  visibility: EdgeVisibility
  historical?: { summary?: string; era?: string }    // NEW
}
```

`resolveOne` maps:
- `{historical:summary}` → `ctx.historical?.summary ?? slot.fallback ?? ''`
- `{historical:era}` → `ctx.historical?.era ?? slot.fallback ?? ''`

The fallback path matters: if the present edge has no linked historical edge, the bridge template's slot resolves to its declared fallback (or empty string), and the spineSummary weaver decides whether to append the bridge clause based on whether the historical context was found.

Default behavior when `ctx.historical` is undefined preserves Phase 3/4 contracts: `{historical:*}` resolves to `''` for any caller that doesn't pass historical context (every caller except the spine-summary path, and even there only when a historical edge is linked).

### `historicalBridge` template authoring

Each present-edge family in `templates/` already declares an empty `historicalBridge: { text: '', expects: {} }`. Phase 5 fills 6 of these with real text. The bridge template is a clause (not a sentence) that prepends to or follows the present-edge `spineSummary`.

Example (CONTROLS → FOUNDED_BY):

```ts
historicalBridge: {
  text: '{subject} founded {object} during {historical:era|the early charters}, and {historical:summary|the original compact} still anchors who answers to whom.',
  expects: { subject: 'properNoun', object: 'properNoun' },
}
```

Templates that need `{historical:*}` slots use the fallback syntax `{historical:era|the early charters}` so a historical-less render still produces well-formed prose. In practice the spine-summary weaver only invokes the historicalBridge template when a historical edge IS linked, so the fallbacks are defensive.

Slot shape note: `{historical:summary}` is a clause/sentence fragment in lowercase (e.g., "the second-wave terraforming failure"); the bridge template should treat it as `'clause'` shape so `reshapeSlot` strips trailing punctuation but does NOT lowercase or strip articles. `{historical:era}` is also lowercase by convention — `'era'` shape skips the article-stripping pass entirely. The `reshapeSlot` helper already accepts both shapes (Phase 1) so no grammarSafety changes are needed.

The 3 historical edge types (`FOUNDED_BY`, `BETRAYED`, `DISPLACED`) get their own template families using the same shape as present types — their `body` templates render the historical summary at attach time; their `spineSummary` and `historicalBridge` fields are unused.

### `renderSpineSummary` weaving

The current `renderSpineSummary` (Phase 3, [`renderSystemStory.ts:90-109`](../lib/generator/graph/render/renderSystemStory.ts)) renders only the top spine edge's `spineSummary` template. Phase 5 changes the function to:

1. Find the top spine edge.
2. Look for a historical edge whose `consequenceEdgeIds` includes the spine edge's id.
3. If found AND the spine edge's family declares a non-empty `historicalBridge` template:
   - Build an `EdgeRenderContext` with `historical: { summary, era }`.
   - Render the `historicalBridge` template using that context.
   - Compose: `<historicalBridge> ` + `<spineSummary>` (or vice versa — implementer's choice; the templates are authored to support whichever comes first).
4. If no linked historical edge OR no bridge template, fall back to Phase 3 behavior — render only `spineSummary`.

The composition order is "historical bridge first, present-tense spine summary second" matching the worked example's "Orison Hold has held the chiral ice belt since the second-wave terraforming failure ... — but the Route Authority and the Kestrel Free Compact can't both set the quota." Implementer may reverse if the templates read better the other way; capture the choice in the commit message.

Cap on weaving: a single historical edge contributes to at most one bridge in spineSummary. If two historical edges are attached and both link to the same top spine edge (rare — should be impossible by Task 6's design), pick the first deterministically.

### Determinism

- `rng.fork('history')` is the new sub-fork for era pool draws and any tie-breaks during attachment. Existing forks (`rules`, `body`, `hooks`) are untouched.
- Spine-edge scoring is deterministic (lookup table, no RNG).
- Subject/object selection is deterministic (taken from the present edge).
- Same seed → same historical edges → same spine summary.

A new RNG fork changes downstream consumption order. The Phase 3 prose-unchanged snapshot does NOT depend on `story` rng output (it pins `narrativeLines` / `narrativeThreads` / settlement / phenomenon prose, not `systemStory`). Verify this assumption holds — if it doesn't, the snapshot baseline changes and Phase 5 will need to regenerate it (capture in the relevant commit message).

### Audit metrics worth tracking after Phase 5

- **Historical edges per system (mean / max):** target 0.5–1.5 mean, max 2.
- **Spine edges with attached history (rate):** target 30–60% of spine edges that score "high" carry a historical edge. Lower rates mean the attachment logic is over-conservative.
- **`spineSummary` length p10/p50/p90:** Phase 4 was 111–116 chars uniformly. Phase 5 expectation: p10 around 110 (sparse, no history), p50 around 150, p90 around 200. Widening is the goal.
- **Spine size p10/p50/p90:** Phase 4 was uniformly 3 (the cap). The Phase 3 carryover note expected this might widen, but spine size is determined by `selectEdges` budget (`selectEdges.spineCap` in Phase 2), not by historical attachment. Re-measure but do not block on widening — historical edges are NOT in `spineEdgeIds`.

Plan exit metric: `spineSummary` length p90 ≥ 150 chars on the deep audit. If this is not met, document the actual value and surface it in the master plan's Phase 7 carryover row.

  ```ts
  export const ERAS = [
  ```

  ```ts
  import type { Confidence } from '../../../types'

    EdgeType, EntityRef, RelationshipEdge,
  } from './types'
  import { mintEdgeId, stableHashString } from './rules'
  import { ERAS, pickEra, type Era } from './data/eras'
  import { templateFor } from './render/templates'
  import { resolveSlots, type EdgeRenderContext } from './render/slotResolver'
  import { capitalizeForPosition, guardDoubledNoun } from './render/grammarSafety'

  type SpineNeedsBackstoryScore = number  // 0.0 (skip), 0.6 (medium), 1.0 (high)

  const NEEDS_BACKSTORY_BY_TYPE: Record<EdgeType, SpineNeedsBackstoryScore> = {
    HOSTS: 0.0,
    WITNESSES: 0.0,
    CONTROLS: 1.0,
    CONTESTS: 1.0,
    DEPENDS_ON: 1.0,
    CONTRADICTS: 1.0,
    DESTABILIZES: 0.6,
    SUPPRESSES: 0.6,
    HIDES_FROM: 0.0,
    FOUNDED_BY: 0.0,
    BETRAYED: 0.0,
    DISPLACED: 0.0,
  }

  const PRESENT_TO_HISTORICAL: Partial<Record<EdgeType, EdgeType>> = {
    CONTROLS: 'FOUNDED_BY',
    CONTESTS: 'BETRAYED',
    DEPENDS_ON: 'DISPLACED',
    DESTABILIZES: 'FOUNDED_BY',
    SUPPRESSES: 'BETRAYED',
    CONTRADICTS: 'BETRAYED',
  }

  const MAX_HISTORICAL_EDGES = 2

  export interface AttachInput {
    spineEdges: RelationshipEdge[]
    rng: SeededRng
  }

  export interface AttachResult {
    historicalEdges: RelationshipEdge[]
  }

  export function attachHistoricalEvents(input: AttachInput): AttachResult {
    const candidates = scoreSpineEdges(input.spineEdges)
    if (candidates.length === 0) return { historicalEdges: [] }

    const historyRng = input.rng.fork('history')
    const out: RelationshipEdge[] = []
    for (const candidate of candidates) {
      if (out.length >= MAX_HISTORICAL_EDGES) break
      const histType = PRESENT_TO_HISTORICAL[candidate.edge.type]
      if (!histType) continue

      const era = pickEra(historyRng)
      const histEdge = mintHistoricalEdge(candidate.edge, histType, era)
      if (histEdge !== null) out.push(histEdge)
    }
    return { historicalEdges: out }
  }

  interface ScoredSpine {
    edge: RelationshipEdge
    score: SpineNeedsBackstoryScore
  }

  function scoreSpineEdges(spine: ReadonlyArray<RelationshipEdge>): ScoredSpine[] {
    const out: ScoredSpine[] = []
    for (const edge of spine) {
      const score = NEEDS_BACKSTORY_BY_TYPE[edge.type] ?? 0.0
      if (score > 0) out.push({ edge, score })
    }
    out.sort((a, b) => {
      if (b.score !== a.score) return b.score - a.score
      // Tie-break by stable hash of edge id, deterministic.
      return stableHashString(a.edge.id) - stableHashString(b.edge.id)
    })
    return out
  }

  function mintHistoricalEdge(
    presentEdge: RelationshipEdge,
    histType: EdgeType,
    era: Era,
  ): RelationshipEdge | null {
    const { subject, object } = pickHistoricalEndpoints(presentEdge, histType)
    if (!subject || !object) return null

    const summary = renderHistoricalSummary({
      subject, object, qualifier: era,
      edgeType: histType, visibility: 'public',
    })
    if (summary === '') return null

    const id = mintEdgeId(`HISTORY:${histType}`, subject.id, object.id, era)
    return {
      id, type: histType,
      subject, object,
      qualifier: era,
      visibility: 'public',
      confidence: 'inferred',
      groundingFactIds: [...presentEdge.groundingFactIds],
      era: 'historical',
      weight: presentEdge.weight * 0.7,
      approxEra: era,
      summary,
      consequenceEdgeIds: [presentEdge.id],
    }
  }

  function pickHistoricalEndpoints(
    presentEdge: RelationshipEdge,
    histType: EdgeType,
  ): { subject: EntityRef | undefined; object: EntityRef | undefined } {
    // Default: carry subject/object from the present edge.
    // Specific historical types may invert depending on the rendering intent.
    if (histType === 'DISPLACED') {
      // DISPLACED: subject = the displaced population (dependent = subject of DEPENDS_ON);
      // object = the resource (object of DEPENDS_ON).
      return { subject: presentEdge.subject, object: presentEdge.object }
    }
    return { subject: presentEdge.subject, object: presentEdge.object }
  }

  function renderHistoricalSummary(ctx: EdgeRenderContext): string {
    const family = templateFor(ctx.edgeType)
    if (family.body.length === 0 || family.body[0].text === '') return ''
    const variant = family.body[0]
    let text = resolveSlots(variant.text, ctx, variant.expects)
    text = capitalizeForPosition(text, 'sentence-start')
    text = guardDoubledNoun(text)
    return text
  }
  ```

  ```ts
  function renderSpineSummary(graph: SystemRelationshipGraph): string {
    const topSpineId = graph.spineEdgeIds[0]
    if (!topSpineId) return ''
    const edge = graph.edges.find(e => e.id === topSpineId)
    if (!edge) return ''
    const family = templateFor(edge.type)
    if (family.spineSummary.text === '') return ''

    const linkedHistorical = findLinkedHistoricalEdge(graph, edge.id)
    const ctx: EdgeRenderContext = {
      subject: edge.subject,
      object: edge.object,
      qualifier: edge.qualifier,
      edgeType: edge.type,
      visibility: edge.visibility,
      historical: linkedHistorical
        ? { summary: linkedHistorical.summary, era: linkedHistorical.approxEra }
        : undefined,
    }

    const bridgeText = (linkedHistorical && family.historicalBridge.text !== '')
      ? renderClause(family.historicalBridge, ctx)
      : ''
    const summaryText = renderClause(family.spineSummary, ctx)

    if (bridgeText === '') return summaryText
    return composeSpineSummary(bridgeText, summaryText)
  }

  function findLinkedHistoricalEdge(
    graph: SystemRelationshipGraph,
    spineEdgeId: string,
  ): RelationshipEdge | undefined {
    for (const edge of graph.edges) {
      if (edge.era !== 'historical') continue
      if (edge.consequenceEdgeIds?.includes(spineEdgeId)) return edge
    }
    return undefined
  }

  function renderClause(template: EdgeTemplate, ctx: EdgeRenderContext): string {
    let result = resolveSlots(template.text, ctx, template.expects)
    result = capitalizeForPosition(result, 'sentence-start')
    result = guardDoubledNoun(result)
    return result
  }

  function composeSpineSummary(bridge: string, summary: string): string {
    // Bridge ends with a separator (',' or '—'); join with a space and lowercase
    // the first char of the summary so it reads as one sentence.
    const lower = summary.length > 0
      ? summary[0].toLowerCase() + summary.slice(1)
      : ''
    return `${bridge} ${lower}`
  ```

## Risks & deferred items

- **Spine size remains uniformly 3.** Per the Phase 3 carryover note, historical edges might "compete for spine slots." In practice they don't: spine is determined by `selectEdges` which only considers present edges. Phase 5 doesn't widen spine size. If wider variance is desired, Phase 7 should tune `selectEdges.spineCap` / its requirement that spine touch 2 layers.
- **`spineSummary` length p90 may not reach 150 chars.** Some systems have spine edges with `historicalBridge.text === ''` (HOSTS, WITNESSES, HIDES_FROM); their spine summaries don't grow. Phase 5 reports the actual p90; Phase 7 may extend HOSTS/WITNESSES with their own bridge templates if needed.
- **Variant selection in `attachHistoricalEvents` always picks `family.body[0]`.** Adequate for Phase 5 since each historical family has 3 variants and one is sufficient. Phase 7 may rotate variants based on stable hash of the present edge id; not blocking.
- **`pickHistoricalEndpoints` carries subject/object directly from the present edge.** This produces semantically odd combinations for some present-edge instances (e.g., `CONTRADICTS:hiddenTruth-vs-publicSurface` whose subject is a settlement and object is a faction → `BETRAYED(settlement, faction)` reads "the settlement broke faith with the faction" which works tonally even if the literal historical event is more nuanced). Phase 7 may add per-present-edge custom endpoint resolution.
- **`{historical:summary}` slot is reserved for templates that want to weave the rendered historical sentence directly.** Task 8's bridge templates don't use it (they only use `{historical:era}`). Future template variants — e.g., a CONTROLS bridge that reads "Since {historical:summary}, {subject} has held {object}" — can use it. The slot resolver supports both regardless.
- **Rule-fire compounding with Phase 4's empty-story rate.** Phase 4 reported 6.77% empty-story rate. Phase 5 may improve this slightly because some empty-story systems gain a non-empty spineSummary via the bridge weave. Re-measure in Task 10 and update the master plan's Phase 7 carryover accordingly.
- **No new RNG forks beyond `'history'`.** `'rules'`, `'body'`, `'hooks'` are unchanged. Task 6's `attachHistoricalEvents` forks `'history'` from the parent rng, which is the same parent fork that `buildRelationshipGraph` already receives. Same seed → same forks → same output.
- **`buildEdgeIndexes` may filter or assume present-only edges.** If it does, Task 7 must update it. Verify before merging Task 7.
